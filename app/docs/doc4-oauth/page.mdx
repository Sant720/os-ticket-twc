# How to set up OAuth Authentication

Intro


## Understanding OAuth 2.0

OAuth 2\.0 is an open standard that allows osTicket to delegate access securely to external identity providers — such as **Microsoft**, **Google**, and **Okta** — instead of handling credentials directly.

### Authentication vs. Authorization

| Concept | Purpose | Example |
|----------|----------|----------|
| **Authentication** | Verifies *who* the user is. | Agents or end users logging in via Google, Microsoft, or Okta. |
| **Authorization** | Grants osTicket permission to access external services (like email servers). | osTicket fetching mail from Gmail using OAuth tokens instead of passwords. |

## Authentication (Single Sign-On)

OAuth 2\.0 Authentication enables **Single Sign-On (SSO)**, allowing **Agents** and **Users** to log in using existing accounts from trusted identity providers.

**Common Providers:**
- Microsoft (Azure AD)
- Google Workspace
- Okta

When configured, users are redirected to the provider’s secure login page.  
After successful sign-in, the provider issues a token verifying the user’s identity, allowing them to access osTicket without needing a local password.

## Authorization (Email Access)

OAuth 2\.0 **Authorization** allows osTicket to connect securely to mail servers — such as Outlook or Gmail — using temporary access tokens.  
This approach replaces legacy “Basic Authentication” (username + password), which many providers have deprecated.

> **Important Notice**  
> Microsoft has deprecated older Outlook endpoints used for Modern Authentication (OAuth2) for **email access**.  
> This does **not** affect user or agent login but can impact token renewal for mail fetching.

If you encounter errors such as:
```
The API version v2 has been depreciated
```
follow the steps in the [Microsoft Authorization Guide](../OAuth2/Microsoft%20Authorization%20Guide) to update your configuration.

### Benefits
- No local password storage  
- Automatic token refresh  
- Compliance with modern security standards (OAuth2 scopes)

### Supported Providers
- Microsoft (Office 365, Exchange Online)
- Google (Gmail, Google Workspace)

## Setting Up the OAuth 2.0 Plugin

Before enabling OAuth Authentication or Authorization, you need to install the official plugin.

### Step 1: Download the Plugin

1. Visit the [osTicket Downloads page](https://osticket.com/download).  
2. Select **v1\.17 or higher**.  
3. Under *Authentication Plugins*, download **Authentication :: OAuth2**.

### Step 2: Install the Plugin

1. Copy the downloaded `.phar` or plugin file to:
   ```
   /include/plugins/
   ```
2. Open osTicket and navigate to:  
   **Admin Panel → Manage → Plugins**

### Step 3: Add and Enable the Plugin

1. Click **Add New Plugin**
2. Install it using the provided interface.
3. Confirm that the plugin appears under the Installed list.
4. Change **Status** from `Disabled` → `Active`, then click **Save Changes**.
